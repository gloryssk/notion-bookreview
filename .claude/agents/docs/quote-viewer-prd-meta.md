# 견적서 웹 뷰 서비스 PRD 메타 프롬프트

## 📋 사용 방법

다음과 같이 입력하면 이 메타 프롬프트가 실행됩니다:

```
"견적서 웹 뷰 + PDF 다운로드 서비스의 PRD를 생성해줘"
```

이 메타 프롬프트는 **prd-generator 에이전트**와 함께 작동하여 완벽한 PRD를 자동 생성합니다.

---

## 🎯 프로젝트 핵심 정보

### 목적
노션에 입력된 견적서 데이터를 클라이언트가 웹 브라우저에서 확인하고 PDF로 다운로드할 수 있는 간단하고 실용적인 서비스를 개발한다.

### 타겟 사용자
견적서를 받는 최종 클라이언트 (공유 링크를 통해 별도 로그인 없이 접근하는 비인증 사용자)

### MVP 핵심 가치
- **빠른 공유**: 노션 데이터를 즉시 웹에서 확인 가능
- **쉬운 다운로드**: 한 번의 클릭으로 PDF 저장
- **반응형 지원**: 모든 디바이스에서 동일한 경험

---

## 🚶 사용자 여정

```
1. 클라이언트가 공유 링크 접속
   ↓ [링크 자동 로드]

2. 견적서 조회 페이지
   ↓ [데이터 렌더링 완료]

   [액션 선택]
   ├─ PDF 다운로드 버튼 클릭 → PDF 다운로드
   └─ 페이지 새로고침 또는 닫음 → 종료

3. PDF 다운로드 (선택사항)
   ↓ [브라우저 다운로드]

4. 완료
   └─ 재접속 또는 종료 가능
```

## ⚡ MVP 기능 명세

### 1. 핵심 기능 (반드시 필수)

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 견적서 데이터 조회 | 노션 API/공유 링크에서 데이터를 가져와 웹에 표시 | 서비스의 핵심 기능 | 견적서 조회 페이지 |
| **F002** | PDF 다운로드 | 현재 보고 있는 견적서를 PDF 파일로 다운로드 | 클라이언트의 주요 니즈 | 견적서 조회 페이지 |
| **F003** | 반응형 레이아웃 | 데스크톱/태블릿/모바일에서 최적화된 화면 표시 | 다양한 기기 지원 필수 | 모든 페이지 |
| **F004** | 데이터 유효성 검증 | 노션에서 가져온 데이터가 정상인지 확인 및 오류 표시 | 데이터 신뢰성 보장 | 견적서 조회 페이지 |
| **F005** | 에러 처리 및 표시 | 링크 만료, 데이터 없음 등 예외 상황 처리 | 사용자 경험 보장 | 견적서 조회 페이지, 에러 페이지 |

### 2. 기술 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F010** | 노션 데이터 연동 | 노션 API 또는 공유 링크에서 데이터 읽기 | 핵심 데이터 소스 연동 | 견적서 조회 페이지 |
| **F011** | 페이지 로딩 상태 | 데이터 로딩 중 UI 상태 표시 | UX 기본 요소 | 견적서 조회 페이지 |

### 3. MVP 이후 기능 (제외)

- 견적서 작성/수정 기능
- 회원가입/로그인
- 견적서 서명 기능
- 실시간 알림
- 다중 언어 지원
- 견적서 템플릿 설정

## 📄 페이지별 상세 기능

### 견적서 조회 페이지

> **구현 기능:** `F001`, `F002`, `F003`, `F004`, `F005`, `F010`, `F011` | **접근:** 공유 링크 직접 접속

| 항목 | 내용 |
|------|------|
| **역할** | 견적서 데이터를 브라우저에 표시하고 PDF 다운로드를 제공하는 메인 페이지 |
| **진입 경로** | 공유 링크를 통해 직접 접속, URL 파라미터에서 견적서 ID 자동 추출 |
| **사용자 행동** | 1. 페이지 접속 → 2. 견적서 자동 로드 → 3. 내용 확인 → 4. (선택) PDF 다운로드 |
| **주요 기능** | • 노션/데이터 소스에서 견적서 데이터 자동 로드 (F010)<br>• 로딩 중 스피너/스켈레톤 UI 표시 (F011)<br>• 데이터 유효성 검증 및 포매팅 (F004)<br>• 견적서 정보 시각적으로 표시 (회사명, 견적 항목, 금액, 날짜 등)<br>• **PDF 다운로드** 버튼 - 클릭 시 현재 페이지를 PDF로 변환 및 다운로드 (F002)<br>• 에러 발생 시 사용자 친화적 에러 메시지 표시 (F005)<br>• 모든 화면 크기(모바일/태블릿/데스크톱)에서 최적화된 레이아웃 (F003) |
| **다음 이동** | • PDF 다운로드 완료 → 페이지 유지<br>• 새로고침 또는 닫음 → 종료<br>• 링크 만료/오류 → 에러 페이지 |

### 에러 페이지

> **구현 기능:** `F005` | **조건:** 데이터 없음, 링크 만료, 서버 오류 시

| 항목 | 내용 |
|------|------|
| **역할** | 오류 상황을 사용자에게 명확히 전달하고 가능한 해결 방법 제시 |
| **진입 경로** | 1) 견적서 조회 페이지에서 데이터 로드 실패 2) 유효하지 않은 URL 직접 접속 3) 링크 만료 상황 |
| **사용자 행동** | 오류 메시지 확인 후 페이지 닫음 또는 링크 발급자에게 문의 |
| **주요 기능** | • 오류 유형별 명확한 메시지 표시<br>  - "찾을 수 없음" (데이터 없음)<br>  - "링크 만료" (시간 초과)<br>  - "서버 오류" (일시적 문제)<br>• 각 오류별 추천 조치 사항 안내<br>• 발급자에게 문의할 수 있는 가이드 제시<br>• 반응형 레이아웃 유지 (F003) |
| **다음 이동** | 페이지 닫음 또는 다시 시도 (새로고침) |

## 🛠️ 기술 스택 (최신 버전)

### 🎨 프론트엔드 프레임워크

- **Next.js 15** (App Router) - React 풀스택 프레임워크, 정적/동적 렌더링 최적화
- **TypeScript 5.6+** - 타입 안전성 및 개발자 경험 향상
- **React 19** - 최신 렌더링 성능 및 동시성 기능

### 🎨 스타일링 & UI

- **TailwindCSS v4** (설정파일 없는 새로운 엔진) - 반응형 유틸리티 CSS
- **shadcn/ui** - 고품질 React 컴포넌트 라이브러리 (Button, Card, Alert 등)
- **Lucide React** - 현대적 아이콘 라이브러리

### 📄 PDF 생성

- **react-pdf** 또는 **html2pdf** - 브라우저에서 PDF 생성 및 다운로드
- **jsPDF** - PDF 파일 생성 라이브러리 (대안)

### 🗄️ 데이터 연동

- **Notion API** (JavaScript Client) - 노션 데이터베이스 및 페이지 데이터 읽기
- **또는** 노션 공유 링크 + 웹 스크래핑 방식 (간단한 데이터 추출용)
- **Axios** 또는 **Fetch API** - HTTP 요청 처리

### 📝 폼 & 검증

- **Zod** - URL 파라미터 및 노션 데이터 스키마 검증

### 🚀 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼

### 📦 패키지 관리

- **npm** - 의존성 관리

## 🔄 정합성 검증 체크리스트 (필수 확인)

PRD 생성 완료 후 반드시 다음을 확인하세요:

### 1단계: 기능 명세 → 페이지 연결 검증

- [ ] 기능 명세의 모든 기능 ID (F001~F011)가 페이지별 상세 기능에 존재하는가?
- [ ] "견적서 조회 페이지"에서 F001~F011이 모두 구현되는가?
- [ ] "에러 페이지"에서 F005가 구현되는가?

### 2단계: 메뉴 구조 → 페이지 연결 검증

- [ ] 메뉴 구조의 모든 페이지 항목이 페이지별 상세 기능에 존재하는가?
- [ ] 메뉴에서 참조하는 모든 기능 ID가 기능 명세에 정의되어 있는가?

### 3단계: 페이지별 상세 기능 → 역참조 검증

- [ ] 각 페이지의 구현 기능 ID가 기능 명세에 정의되어 있는가?
- [ ] 모든 페이지가 메뉴 구조에서 접근 가능한가?

### 4단계: 누락 및 고아 항목 검증

- [ ] 기능 명세에만 있고 페이지에서 구현되지 않은 기능이 있는가?
- [ ] 페이지에만 있고 기능 명세에 정의되지 않은 기능이 있는가?
- [ ] 메뉴에만 있고 실제 페이지가 없는 항목이 있는가?

## 🎯 최종 PRD 작성 체크리스트

생성된 PRD가 다음을 모두 포함하는지 확인하세요:

- [ ] **프로젝트 핵심**: 목적(1줄) + 타겟 사용자(1줄)
- [ ] **사용자 여정**: 페이지 간 이동 흐름 (링크 접속 → 조회 → 다운로드 → 종료)
- [ ] **기능 명세**: F001~F011 전체 7개 기능, 각 기능별 관련 페이지 명시
- [ ] **메뉴 구조**: 공유 링크 기반 구조 (인증 메뉴 불필요)
- [ ] **페이지별 상세**: 2개 페이지 (조회 + 에러)
  - 각 페이지: 역할, 진입 경로, 사용자 행동, 주요 기능, 다음 이동
  - 각 페이지: 구현 기능 ID 명시
- [ ] **데이터 모델**: Quote + QuoteItem 최소 구조
- [ ] **기술 스택**: Next.js 15, React 19, TailwindCSS v4, PDF 라이브러리, 노션 연동 기술
- [ ] **URL 경로**: 사용하지 않음 (페이지 이름만 사용)
- [ ] **정합성**: F001~F011이 모두 페이지에서 구현되는지 확인
