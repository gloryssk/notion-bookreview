# 견적서 웹 뷰 서비스 PRD 메타 프롬프트 사용 가이드

## 빠른 시작

### 1단계: 메타 프롬프트 실행

다음 명령어를 Claude에 입력하기만 하면 됩니다:

```
견적서 웹 뷰 + PDF 다운로드 서비스의 PRD를 생성해줘
```

### 2단계: 자동으로 생성되는 완벽한 PRD

메타 프롬프트는 다음을 모두 자동 생성합니다:

- ✅ 프로젝트 핵심 정보 (목적, 타겟 사용자)
- ✅ 전체 사용자 여정 플로우
- ✅ 7개의 기능 명세 (F001~F011)
- ✅ 메뉴/네비게이션 구조
- ✅ 2개 페이지 상세 기능 (조회 + 에러)
- ✅ 데이터 모델 (Quote, QuoteItem)
- ✅ 최신 기술 스택 명시 (Next.js 15, React 19 등)

---

## 📋 메타 프롬프트에 포함된 요소

### 프로젝트 정의

| 항목 | 내용 |
|------|------|
| **목적** | 노션 견적서를 웹에서 조회/PDF 다운로드 가능하게 만드는 서비스 |
| **타겟 사용자** | 견적서를 받는 최종 클라이언트 (비인증 사용자) |
| **핵심 가치** | 빠른 공유, 쉬운 다운로드, 반응형 지원 |

### 사용자 여정

```
공유 링크 접속 → 견적서 조회 → PDF 다운로드 → 완료
```

### 기능 목록 (7개, MVP 필수)

| 기능 ID | 기능명 | 설명 |
|--------|--------|------|
| **F001** | 견적서 데이터 조회 | 노션에서 데이터 로드하여 표시 |
| **F002** | PDF 다운로드 | 현재 견적서를 PDF로 저장 |
| **F003** | 반응형 레이아웃 | 모든 디바이스 최적화 |
| **F004** | 데이터 유효성 검증 | 데이터 신뢰성 확보 |
| **F005** | 에러 처리 | 예외 상황 처리 |
| **F010** | 노션 데이터 연동 | 노션 API/공유 링크 연동 |
| **F011** | 페이지 로딩 상태 | 로딩 UI 표시 |

### 페이지별 구현

**2개 페이지만 필요:**

1. **견적서 조회 페이지** → F001, F002, F003, F004, F005, F010, F011 모두 구현
2. **에러 페이지** → F005 (에러 표시) 구현

### 데이터 모델

**Quote (견적서)**
- id, title, companyName, clientName, quotationDate, validUntilDate
- items (QuoteItem 배열), totalAmount, currency, notes

**QuoteItem (견적 항목)**
- id, description, quantity, unitPrice, subtotal

### 기술 스택

- **프레임워크**: Next.js 15 + React 19 + TypeScript 5
- **스타일링**: TailwindCSS v4 + shadcn/ui
- **PDF**: react-pdf 또는 html2pdf
- **데이터**: Notion API + Zod
- **배포**: Vercel

## 🎯 메타 프롬프트의 장점

### 1. 완전한 정합성 보장

```
기능 명세 ↔ 메뉴 구조 ↔ 페이지 상세 기능이 완벽하게 동기화됨
```

- F001~F011 모든 기능이 페이지에서 구현됨
- 메뉴의 모든 항목이 실제 페이지로 매핑됨
- 누락되거나 고아 항목이 없음

### 2. MVP 핵심만 포함

- 필수 기능만 7개 (조회 + 다운로드 + 지원)
- 불필요한 기능 제외 (로그인, 서명, 알림 등)
- 1인 개발자도 충분히 구현 가능한 범위

### 3. 최신 기술 스택

- Next.js 15 (최신)
- React 19 (최신)
- TailwindCSS v4 (최신 방식)

### 4. 즉시 개발 가능

PRD만으로도 바로 코딩 시작 가능:
- 페이지 구조 명확
- 기능 목록 구체적
- 데이터 모델 정의됨
- 기술 선택 완료

## 💡 핵심 포인트

### 이 메타 프롬프트가 보장하는 것

1. **완벽한 정합성**: 기능 ↔ 페이지 ↔ 메뉴가 완전히 동기화
2. **MVP 집중**: 불필요한 기능은 모두 제외
3. **최신 스택**: Next.js 15, React 19, TailwindCSS v4
4. **1인 개발자 친화**: 6주~3개월 안에 완성 가능한 범위
5. **즉시 개발 가능**: PRD만으로도 코딩 시작 가능

### 메타 프롬프트의 구성

```
프로젝트 핵심
  ↓
사용자 여정
  ↓
기능 명세 (7개, MVP)
  ↓
메뉴 구조
  ↓
페이지별 상세 기능 (2개 페이지)
  ↓
데이터 모델
  ↓
기술 스택 (최신 버전)
```

## 📞 FAQ

### Q: 이 메타 프롬프트를 어떻게 실행하나?

```
보조 모델(Claude)에 다음 입력:
"견적서 웹 뷰 + PDF 다운로드 서비스의 PRD를 생성해줘"
```

### Q: 생성되는 PRD가 정말 완벽한가?

맞습니다. 이 메타 프롬프트는:
- prd-generator의 정합성 원칙 완전 준수
- 기능-페이지-메뉴 3중 동기화 보장
- 검증 체크리스트 포함

### Q: 기능을 추가하려면?

메타 프롬프트 수정:
1. 기능 명세에 새 기능 추가 (F012, F013 등)
2. 메뉴 구조에 매핑
3. 페이지별 상세 기능에 구현 방법 기술
4. 정합성 검증 실행

### Q: 기술 스택을 바꾸려면?

메타 프롬프트의 "기술 스택" 섹션 수정:
- PDF 라이브러리 변경 (html2pdf, jsPDF 등)
- 노션 연동 방식 변경
- 다른 UI 프레임워크 사용
