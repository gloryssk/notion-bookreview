## 🏗️ 메타 프롬프트 구조

### 1. 프로젝트 핵심 (3줄)

```
목적: 노션 견적서를 웹에서 조회/PDF 다운로드 가능하게 함
타겟: 견적서를 받는 최종 클라이언트 (비인증)
가치: 빠른 공유, 쉬운 다운로드, 반응형 지원
```

### 2. 사용자 여정 (4단계)

```
1. 공유 링크 접속
   ↓
2. 견적서 조회 페이지 (데이터 자동 로드)
   ↓
3. PDF 다운로드 (선택사항)
   ↓
4. 완료
```

### 3. 기능 명세 (7개, MVP)

| F# | 기능명 | 설명 |
|----|--------|------|
| **F001** | 견적서 데이터 조회 | 노션에서 데이터 로드 |
| **F002** | PDF 다운로드 | 견적서를 PDF로 저장 |
| **F003** | 반응형 레이아웃 | 모든 디바이스 최적화 |
| **F004** | 데이터 유효성 검증 | 데이터 신뢰성 확보 |
| **F005** | 에러 처리 | 예외 상황 처리 |
| **F010** | 노션 데이터 연동 | 노션 API/공유 링크 |
| **F011** | 페이지 로딩 상태 | 로딩 UI 표시 |

### 4. 페이지 구조 (2개 페이지)

```
📄 견적서 조회 페이지 (메인)
   └─ F001, F002, F003, F004, F005, F010, F011 구현

⚠️ 에러 페이지 (조건부)
   └─ F005 구현
```

### 5. 데이터 모델 (2개 테이블)

```
Quote (견적서)
├─ id, title, companyName, clientName
├─ quotationDate, validUntilDate
├─ items: QuoteItem[]
└─ totalAmount, currency, notes

QuoteItem (항목)
├─ id, description
├─ quantity, unitPrice
└─ subtotal
```

### 6. 기술 스택 (최신 버전)

```
Frontend: Next.js 15 + React 19 + TypeScript 5
Styling: TailwindCSS v4 + shadcn/ui + Lucide React
PDF: react-pdf 또는 html2pdf
Data: Notion API + Zod
Hosting: Vercel
```

## ✅ 메타 프롬프트의 장점

### 1. 완벽한 정합성 보장

```
기능 명세(F001~F011)
    ↓ 1:1 매핑
메뉴 구조
    ↓ 1:1 매핑
페이지별 상세 기능
    
→ 누락/고아 항목 0개 보장
```

### 2. MVP 명확하게 정의

```
포함: 조회 + 다운로드 + 기본 기능
제외: 로그인, 작성, 서명, 알림 등
→ 1인 개발자 6주~3개월 완성 범위
```

### 3. 최신 기술 스택 자동 적용

```
Next.js 15 (최신 App Router)
React 19 (최신 동시성 기능)
TailwindCSS v4 (설정파일 없는 방식)
→ 2025년 모던 스택 보장
```

### 4. 즉시 개발 가능

```
PRD만으로도:
- 폴더 구조 결정 가능
- 컴포넌트 설계 가능
- 데이터 모델 정의 가능
- 라이브러리 선택 완료
```

## 🔄 정합성 검증 (메타 프롬프트 포함)

### 4단계 검증 프로세스

```
1단계: 기능 명세 → 페이지 연결 검증
       F001~F011 모두 페이지에서 구현되는가?

2단계: 메뉴 구조 → 페이지 연결 검증
       모든 메뉴 항목이 실제 페이지에 매핑되는가?

3단계: 페이지 → 역참조 검증
       모든 페이지가 메뉴에서 접근 가능한가?

4단계: 누락 확인
       고아 항목이나 누락된 항목이 없는가?
```

### 자동 검증 체크리스트 포함

메타 프롬프트에 25개 항목 검증 리스트 포함:
- 기능 명세 3개 항목 ✓
- 메뉴 구조 3개 항목 ✓
- 페이지 검증 3개 항목 ✓
- 누락 확인 4개 항목 ✓

## 🎓 메타 프롬프트 설계 원칙

### 1. prd-generator 가이드라인 완전 준수

```
✓ 절대 생성하지 말 것 항목 준수
  (개발 우선순위, API 라우트, 인프라, 마일스톤 제외)

✓ 반드시 생성할 것 항목 포함
  (핵심 정보, 사용자 여정, 기능 명세, 메뉴, 페이지, 데이터, 스택)

✓ 기능 정합성 원칙 3중 검증
  (기능-메뉴, 기능-페이지, 메뉴-페이지)

✓ 최신 기술 스택 명시
  (Next.js 15, React 19, TailwindCSS v4)
```

### 2. 기능 정합성 보장 원칙 (CRITICAL)

```
모든 섹션 상호 참조:
1. 기능 명세의 모든 기능 → 메뉴 구조에서 구현
2. 메뉴 구조의 모든 항목 → 페이지별 상세 기능에서 구현
3. 페이지별 상세 기능 → 기능 명세에서 정의
4. 누락 금지: 한 섹션에만 있는 항목 0개
5. 중복 방지: 같은 기능이 여러 페이지에 분산 안 함
```

### 3. MVP 중심 설계

```
필수만 포함: 조회 + 다운로드 + 기본 지원
최소 데이터 모델: Quote + QuoteItem만
최소 페이지: 2개 (조회 + 에러)
최소 기능: 7개 (핵심 5개 + 지원 2개)
```

### 4. 1인 개발자 친화

```
범위: 6주~3개월 완성 가능
복잡도: 중간 이하
```
